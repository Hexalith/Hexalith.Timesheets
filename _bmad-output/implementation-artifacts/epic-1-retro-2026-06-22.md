# Epic 1 Retrospective (Re-run): Trusted Time Capture & Activity Governance

## Why This Retrospective Exists

Epic 1 already had a retrospective on 2026-06-19, when it contained nine stories (1.1-1.9). On
2026-06-20 a correct-course readiness assessment found that several epics had been declared
story-complete while real launch-readiness prerequisites were still missing or deferred into a
catch-all Epic 5. The repair added two stories **inside Epic 1** — Story 1.10 (Work Reference
Validation Adapter) and Story 1.11 (Command Performance Evidence) — and both were completed on
2026-06-22. This retrospective re-reviews the **full eleven-story Epic 1** and supersedes the
2026-06-19 record. It runs autonomously in YOLO mode; Jerome is represented by sprint status, story
records, review notes, source code, and planning artifacts.

The single most important lesson is named up front so it frames everything below:

> **Story-complete is not launch-ready.** The 2026-06-19 retro closed Epic 1 against story-level
> review approval. The readiness check the next day found that Story 1.7's Work-reference path had no
> concrete validator, and that NFR10 command-performance evidence had been pushed entirely into Epic
> 5. Both gaps were real, both were fixable inside Epic 1, and neither had been surfaced by the first
> retrospective.

## Key Learnings

1. **Story-complete is not launch-ready.** A retrospective must cross-check epic-level user-value
   delivery, not only story-level review approval. If a story says a path is "gated on a future
   decision" or evidence is "reserved for later," that is a launch-readiness flag, not a footnote.
2. **Deferred prerequisites belong in the owning epic, not a downstream hardening bucket.** Epic 5 was
   absorbing work (Works adapter, NFR evidence, magic-link loader, export preview) that earlier epics
   needed to honestly claim their value. The repair rehomed it: 1.10/1.11 in Epic 1, 3.6/3.7 in Epic
   3, 4.8/4.9/4.10 in Epic 4. Epic 5 is now verification-only.
3. **When a direct dependency drags a conflicting transitive closure, introduce a light port.**
   `IWorksQueryChannel` mirrors `IDomainQueryInvoker` but depends only on EventStore *Contracts*; the
   host binds the real invoker. This is now the reference pattern for cross-host integration (reusable
   by Story 4.8).
4. **Verify the running infrastructure claim before you design on it.** The "Works has no consumer
   query" note was stale. Critical infrastructure assertions in planning docs must be verified against
   the actual submodule at dev time.
5. **Measure the real composed path and assert the real result.** The NFR10 lane drives the in-process
   command services and asserts the acknowledgement outcome each iteration. Percentile math is testable
   infrastructure and lives in the fast baseline; the lane itself stays opt-in (`TIMESHEETS_PERF=1`,
   `Assert.Skip`) with a fitness test guarding the gate.
6. **Record deferred verdicts honestly.** The EventStore wire-path NFR10 measurement is *waived/
   deferred* with a rationale, so Epic 5 aggregates rather than re-measures — and nobody mistakes the
   in-process number for wire-path proof.
7. **File-List honesty is a hard control, and it has not stuck.** Two repair stories still shipped with
   omitted files and stale counts that review had to fix. This needs an enforcement mechanism, not
   another good intention.
